# CI構築 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PR作成時・mainへのpush時に `flutter analyze` / `dart format` / `flutter test` を自動実行するGitHub Actionsワークフローを構築し、mainブランチの必須ステータスチェックとして組み込む。

**Architecture:** `.github/workflows/ci.yml` に単一ジョブ `flutter-ci` を定義する。ワークフローがmainで最低1回実行された後、GitHub REST APIでブランチ保護ルールに組み込む。

**Tech Stack:** GitHub Actions, `subosito/flutter-action@v2`, Flutter 3.44.4 (stable) / Dart 3.12.2

## Global Constraints

- 新規依存パッケージの追加はしない(Actions側のみの変更)。
- Edge Functions(Deno)のCIは対象外(Issue #67で別対応)。
- 依頼されていないファイルは変更しない。

### Task 1: CIワークフローの作成

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: なし
- Produces: GitHub Actions上のジョブ名 `flutter-ci`(Task 2のブランチ保護設定で参照する)

- [ ] **Step 1: ci.ymlを作成**

`.github/workflows/ci.yml` を新規作成:
```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  flutter-ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.44.4'
          channel: 'stable'

      - name: Install dependencies
        run: flutter pub get

      - name: Check formatting
        run: dart format --output=none --set-exit-if-changed .

      - name: Analyze
        run: flutter analyze

      - name: Run tests
        run: flutter test
```

- [ ] **Step 2: YAML構文を確認**

Run: `cat .github/workflows/ci.yml`
Expected: 上記内容が正しく保存されていること(インデント崩れがないか目視確認)。

- [ ] **Step 3: コミット**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions workflow for flutter analyze/format/test"
```

- [ ] **Step 4: push してPRを作成**

```bash
git push -u origin feature/ci-setup
```

`.github/PULL_REQUEST_TEMPLATE.md` に従い `gh pr create` でPRを作成する。本文に `Closes #50` を含める。

- [ ] **Step 5: PRのCIが成功することを確認**

Run: `gh pr checks <PR番号> --watch`
Expected: `flutter-ci` ジョブが `pass` になること。失敗した場合はログを確認し原因を修正する(ワークフロー自体の記述ミスか、analyze/format/testの実コード起因かを切り分ける)。

**Interfaces:**

  -F "restrictions=null"
